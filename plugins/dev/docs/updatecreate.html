Creating an IVC Update Site
Author: James Ellis
Last Modified: April 18, 2005

Summary

For developers external to the IVC development group, distributing new features requires creating your own Update Site and hosting your features there. This will allow IVC users to acquire new plug-ins and updates using the Software Update Tool just as they do with plug-ins from the IVC update site.

Pre-requisites

Before you read this document, it is useful to have some working knowlege of Eclipse and the Eclipse plug-in framework. We will be discussing the creation of Eclipse Update Site projects in particular, but no prior knowledge of this topic is assumed.

Creating an Update Site

Creating an IVC Update Site is a relatively straightforward process. These sites have three primary pieces: a site map (site.xml), .jar files for the plug-ins that are included in the features in a "plugins" directory, and .jar files for the features themselves in a "features" directory. To create the Update Site project in Eclipse, go to File->New->Project... and select Update Site Project from the Plug-in Developement section.

Fig1: Select Update Site Project from the Plug-in Development section.

Give the Update Site project a name and press Finish, and the site.xml and directories mentioned above are created. This is really all it takes to create the site, but to set up the structure of how the features will be displayed requires manipulating the site.xml file. Looking at the Features tab of the site.xml provides the means to add new features and organize these features into categories. To add a new category, select New Category... and give it a name, label, and description. This allows you to group features into several different groups if desired, but is not required.

Fig2: The site.xml editor's Feature tab is where to add features and create new categories. (Click to Expand)

Adding features to the site is discussed at length in the article about creating features and adding to an existing update site, Extending the IVC Update Site, So we will not go further into that here. Once the features have been added and categorized, this site.xml and the two directories and their contents should be moved to the site location, in the exact same directory structure. For example, if you had a directory on a site such as:

http://mysite.org/ivcplugins/

you could add a subdirectory called update perhaps, and then have a resulting directory structure like: 

http://mysite.org/ivcplugins/update/site.xml
http://mysite.org/ivcplugins/update/plugins/(plugin .jar files)
http://mysite.org/ivcplugins/update/features/(feature .jar files)

and the resulting URL for the update site would be: http://mysite.org/ivcplugins/update/
