Using the IVC Software Update Tool
Author: James Ellis
Last Modified: April 18, 2005

Summary

The Software Update Tool in IVC provides a means for users to acquire new plug-ins, update existing plug-ins, and manage their plug-in configuration in a simple manner. The tool allows for a well managed distribution system with versioning that prevents the user from having to manually download and install features and makes explicit any dependencies of a plug-in to make the update process as painless as possible. For these reasons, this is the means of choice for distributing new plug-ins or updates to existing plugins for IVC.

Getting New Features

The first topic to cover with the Software Update Tool is how to use the provided tool in the IVC distribution to update your plug-in configuration. When IVC is first launched, the Help menu will contain the necessary menu items to use the tool.

Fig1: The IVC Help menu. Notice the menu items for Update... and Manage Configuration... which are used to launch the Software Update Tool utilities.

First, select Update... from the Help menu. This will open up the Install/Update dialog. We will first look at how to get new features from an existing update site, so select "Search for new features to install" and press Next >.

Fig2: The Install/Update dialog is where you specify whether you want to look for new features or update existing features.

In the next dialog, we must specify where we want to look for new features. If this is the first time the tool is run, there will be no sites listed. Click the New Remote Site... button and a dialog will appear asking for a name and URL. We will add the IVC update site, so put in a name such as IVC Update Site and the URL: http://ivc.sourceforge.net/update and press OK.

Fig3: Specify a name and insert the URL in the figure to add the IVC Update Site to the Update tool.

Now that we have a site, press the '+' to the left of the name of the newly added site to expand the tree view of what categories of items are available. For this demonstration we will choose IVC Plug-ins and press Next >. The tool will then search the site for items in that category, and list whatever is available that we do not already have in the next dialog.

Fig4: The tool finds all available plug-ins in the categories specified. This figure shows an example of what this might look like for a particular configuration, results will vary.

We can now simply pick one of the plug-ins that we want to add to our configuration. If a dependency is not met, the tool will signify next to that plug-in when we select it that we need to also select a dependency if we wish to use that plug-in. By highlighting a plug-in and pressing the Properties button, more information about the plug-in can be obtained. After selecting one of the plug-ins, press Next > to continue the installation. Mark your acceptance of the terms of the license agreement for the plug-in and press Next >. Finally, verify the installation location is where you desired, this should be the installation location of IVC, and press Finish to install. The tool may ask about jar verification if the feature has not been digitally signed.

The tool will complete the installation and then ask if you wish to restart the application for changes to take effect. It is best to say Yes here, and then simply relaunch IVC to use the new plug-in. You should notice new menu items corresponding to the features that were added. Some plug-ins will also have corresponding Sample Data plug-ins that are available on the update site to provide some input files that can be loaded in IVC with a Persister to demonstrate the functionality of a plug-in.

Updating Existing Features

When a feature has been added to IVC in the past, but the developer has made an update available in the form of a new version of the feature, the Software Update Tool can be used to perform this upgrade in a standardized fashion. To do so, we select the same menu item as above, Update... from the Help menu. This will open up the Install/Update dialog. Now select "Search for updates of the currently installed features" and press Next >.

Fig5: To search for updates of existing features, select "Search for updates of the currently installed features".

The tool will now scan all of the installed features to look for updates. The next screen will show the results. It should be noted that the results are based somewhat on the prefences specifed in the File->Preferences dialog of IVC, in the Install/Update category. The Valid updates section determines whether only service updates of the same primary version, i.e. an update from 1.0.1 to 1.0.2, are the only updates to allow, or whether minor incremental updates should also be allowed, such as an update from 1.0.9 to 1.1.0. This is really dependent on the actual developer of the feature and what they have changed, but in general it should be considered safe to allow minor incremental updates as long as all other dependencies are met.

If a desired update is found, the process of upgrading is nearly identical to that of aquiring a new feature. Select the updates that are desired and proceed through the installation as described in the section above.

Managing the Configuration

The update tool allows a means for managing the current plug-in configuration from inside of the IVC application as well. By selecting Manage Configuration... in the Help menu of IVC, a dialog will show the current plug-in configuration.

Fig6: An example IVC plug-in configuration. (Click to Expand)

This dialog allows you do perform various activites on the selected features. If you wish to uninstall, or just simply disable, but not completely remove a feature, this dialog allows for this choice. This dialog also has a quick link to scan for updates of a selected feature, as well as a link to look at the properties defined for that feature.
