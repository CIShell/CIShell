Glossary of Technical Terms

Authors: James Ellis, Bruce Herr
Last Modified: April 20, 2005

A

Algorithm : implementation of the Algorithm.java interface, does any sort of computation in its execute() method, typically on a DataModel, often producing another DataModel as a result.

B

C

Console : interface between IVC and the user for messaging purposes, plug-ins can write messages to the Console at various severity levels.

Core Plug-in : minimalist IVC core, which defines an interface for plug-ins to extend from and handles crucial runtime responsibilities to manage the system.

Core Feature : contains seven primary IVC plug-ins that combine to create the GUI application in a seperable, modular framework.

D

DataModel : some data structure containing information that was either generated or loaded from outside IVC using a Persister, IVC allows total flexibilty of DataModel format, with the ability to add new model types when desired.

E

F

Feature : the unit of distribution of IVC plug-ins with the Update Site and Software Update Tool. Plug-ins are grouped into features, since often a singular plug-in is too small of a unit and a grouping needs to be distributed. It is also often the case though that a feature consists of a single plug-in, for less complex IVC plug-ins.

H

I

IVC : Information Visualization Cyberinfrastructure

J

K

L

Library Plugin : a plug-in that does not necessarily contribute to the immediate functionality of IVC, but adds some libraries, such as .jar files that other plug-ins can utilize by having the library plug-in as a dependency in their plugin.xml file.

Logging : IVC logs information to two seperate files at locations determined by the user in preferences, an error log with problems that have arose during runtime, and a user log with information about what occured during the session.

M

Messaging : IVC has a generalized messaging framework which allows plug-ins to request messages be sent, and then delegate to any such messaging plug-ins that are capable of interacting with the user. One implemenation used is GUI based with dialogs, but this could be interchanged with something totally command line based for example.

ModelManager : keeps track of the DataModels that have been generated or loaded into IVC. A DataModel View graphically displays this information to the user and provides an interface to manipulate the internal state of this ModelManager.

N

O

P

Persister : has the ability to save and restore DataModels to and from defined file formats. Every Persister specifies exactly what file formats it applies to, and what DataModels it can be used with.

Plugin : adds functionality to the IVC Application, plug-in based architecture allows flexibilty in adding new features and modifying configuration without having a tangled web of dependencies. The common categories of IVC Plug-ins are: Preprocessing, Modeling, Analysis, Visualization, and Interaction.

Property : several Property classes are defined in the IVC core, such as PluginProperty, AlgorithmProperty, DataModelProperty, and PersisterProperty - these classes define various attributes that implementations should provide about themselves to be used to provide proper functionality in IVC.

Q

R

RCP : the Eclipse Rich Client Platform, this is a term for the framework the IVC application is built upon, using an Eclipse core foundation and plug-in framework with a customized application layer built on top of it. [optional reference: Rich Client Tutorial Part 1]

S

Sample Data Plugin : a plug-in that simply contributes some data files that can be used to show the functionality of a plug-in. These sample data plug-ins are typically dependent on the plug-in that they provide sample data for (i.e. list the plug-in as a dependency in the plugin.xml file).

Scheduler : maintains a queue of Algorithmss waiting to be processed and allows a specified number of them to be run simultaneously. Algorithms can also be scheduled for specific date/time in the future.

Software Update Tool : uses the Eclipse Update framework to provide a standardized means of plug-in feature distribution. Internal to the IVC application, the tool allows users to connect to update sites and look for new features, upgrade existing features, or manage the current configuration. [optional reference: How to Keep Up to Date (eclipse.org)]

T

U

Update Site : IVC update sites are the preferred means of distribution of new plug-ins and updates to existing plug-ins. These sites connect with IVC through the Software Update Tool and display the available plug-in features that the user can acquire. These features can be grouped into categories and all dependencies are managed by the features themselves and the tool. [optional reference: How to Keep Up to Date (eclipse.org)]

V

Visualization : shows some graphical display of a DataModel(s) which allows the user to more easily interpret or understand the data.
