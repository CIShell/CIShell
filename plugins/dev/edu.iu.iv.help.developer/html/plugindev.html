Beginning Plug-in Development
Author: James Ellis
Last Modified: April 20, 2005

Summary

Developing plug-ins for IVC using the Eclipse PDE (Plug-in Development Environment) that can be used in the RCP (Rich Client Platform) IVC framework based off of a simple template can greatly simplify the learning curve of a developer wishing to contribute a new plug-in to IVC. This tutorial will guide a new developer in setting up their development environment to use the other tutorials provided to aid in IVC plug-in developement.

Pre-requisites

Before you read this document, it is useful to have some working knowlege of Eclipse and the Eclipse plug-in framework, but no prior knowledge of this topic is assumed.

Getting Eclipse

The first step in developing your new IVC plug-in will be to download and install the Eclipse IDE with PDE (Plug-in Development Environment) included. Of course, if you already have Eclipse installed this section can be skipped. PDE has been included with Eclipse to some extent since version 2.1, but this tutorial will be based off of the 3.0.1 version.

Downloading Eclipse

The latest releases of Eclipse can always be found at http://www.eclipse.org/downloads/index.php This page should look something like the following:

Fig1: The Eclipse.org download site. (Click to Expand)

From this page you can select one of the mirrors listed, or simply scroll to the very bottom where it says "Main Eclipse Download Site" (or just click the previous link) to go to the main download site if you don't want to bother with a mirror. Now you should be confronted with the following page where you can select exactly what you wish to download:

Fig2: The available Eclipse releases for download. (Click to Expand)

We will select the Latest Release, the first item listed in the top section to download version 3.0.1. This will bring you to a page to allow you to select the platform you wish to download Eclipse for (in the Eclipse SDK section), where you can select whichever operating system you are using. The download will now either begin, or possibly involve selecting one of the mirrors listed if you are still required to select a mirror. This download may take a little time since it is rather large, but it will be worth it.

Trouble? Eclipse does not include a Java runtime environment (JRE). You will need a 1.4.1 level or higher Java runtime or Java development kit (JDK) installed on your machine in order to run Eclipse.

Installation

Once the file has finished downloading, it is a matter of extracting the files from whatever type of archive is downloaded (.zip typically) to whatever location is desired (usually C:\eclipse in Windows). This will result in the new directory structure that looks something like this:

Fig3: The Eclipse install directory. (Click to Expand)

The important thing to find is the eclipse.exe (or equivalent executable on other platforms) that you can now double-click on to launch Eclipse for the first time. Just select 'ok' to the defaults about workspace location unless you have a preference. And then when confronted with the following welcome screen, press the button in the top right that says Go to the workbench (unless you wish to browse around at the other options to learn more, they are always available through Help->Welcome.

Fig4: The Eclipse Welcome Screen. (Click to Expand)

That's it, Eclipse is now up and running and we are ready to move on to the job of actually creating plug-ins using the Plug-in Development Environment and the custom IVC templates. First we have to get the IVC Plug-in Development feature from the IVC Update Site.

Getting the IVC Plug-in Development Feature

To get the IVC Plug-in Development Feature from the IVC Update Site, we will use the Eclipse Update Manager tool from within Eclipse. To do this, go to Help->Software Updates->Find and Install... in the main menu of Eclipse. In the dialog that comes up, choose Search for new features to install and press Next >. Now we want to add the IVC Update Site to the list of sites in the dialog, so select New Remote Site... and give it a name and the URL: http://ivc.sourceforge.net/update/ and press OK.

Fig5: Add the IVC Update Site.

Now press the '+' next to the name that was given to the site to expand the tree of available categories to download from the IVC Update Site. Select the IVC Plug-in Development category and press Next > at the bottom of the screen and the tool will search for the selected features.

The tool should find the IVC Plug-in Development feature available for download. Select this item and press Next > to continue.

Fig6: Selecting the IVC Plug-in Development Feature.

Mark your acceptance of the licensing agreement, press Next >, and then validate the install location and press Finish to start installing the Plug-in Development feature. When asked about jar verification for an unsigned feature, just press Install to continue. This will install the IVC Plug-in Development templates that are used in the other tutorials in this guide to aid plug-in development as well as the underlying IVC core plug-ins that are needed for IVC development to plug into the core. This is not an exhaustive set though, and it may be possible that something that you wish to extend from is not included. In this case, you will have to reconnect to the IVC Update Site and browse the available plug-ins to find what you need. The Plug-in Development feature should have everything you need though for stardard plug-in development based on the items included in the core feature of IVC.

Restart Eclipse when asked, and continue on to one of the other tutorials for specific information about using one of the included templates.
